1. Key Derivation Hierarchy

Overview: Quantum Armor Vault (QAV) uses a hierarchical key derivation system to manage cryptographic keys from the user's password. The system employs Argon2id for key derivation and HKDF for key separation, ensuring that compromise of one key does not affect others.

Key Hierarchy Structure

Cryptographic Parameters

Component Algorithm Parameters Output Size
Password → KEK Argon2id Memory: 64 MiB, Iterations: 3, Lanes: 4 32 bytes
KEK → MEK XChaCha20-Poly1305 24-byte nonce (random) 64 bytes
MEK → Subkeys HKDF-SHA256 Context: purpose string 32 bytes each
Shamir Sharing Shamir's Secret Sharing Threshold: 3, Total shares: 5 64 bytes reconstructed

2. Vault Format V4

Overview: The V4 vault format is a carefully structured binary format that stores all encrypted data in a single file. It includes cryptographic verification, integrity protection, and metadata blocks for user identity and recovery.

Byte Layout

Format Details

  • Magic Number: "USBVLT04" identifies the vault format version and enables format detection
  • Salt: 32-byte cryptographically random value used in Argon2id, prevents rainbow table attacks
  • Verification: Plaintext marker encrypted to verify correct password derivation
  • Wrapped MEK: Master Encryption Key encrypted with KEK + HMAC, ensures integrity
  • State Version: 64-bit counter for rollback protection, incremented on each change
  • Encrypted Index: All file metadata encrypted with Index Encryption Key

3. Authentication Flow (SRP-6a)

Overview: Quantum Armor Vault uses SRP-6a (Secure Remote Password) to authenticate users without transmitting passwords. The KEK is derived from the password and used to verify the user's identity to the server.

SRP-6a Protocol Exchange

Security Properties

Property Description
Zero-Knowledge Proof Server never sees the password or KEK, only the SRP verifier hash
Mutual Authentication Both client and server prove knowledge of the shared secret (M1 and M2)
Perfect Forward Secrecy SRP session keys are ephemeral and not reused across sessions
Strong Password Hashing Argon2id with 64 MiB memory makes brute force attacks expensive
Replay Protection JWT tokens include timestamp and device fingerprint claims

4. Sharing Protocol (Sealed Box)

Overview: Quantum Armor Vault uses public-key cryptography (X25519) for encrypted sharing without key exchange. The protocol is based on the sealed box pattern, ensuring forward secrecy and authentication. Optional post-quantum ML-KEM-1024 support provides hybrid quantum-resistant encryption.

6. Device Enrollment Flow

Overview: New devices must be enrolled in the vault system before they can access vaults. This prevents unauthorized devices from gaining access and allows administrators to control which devices are trusted.

Device Enrollment Process

Device Fingerprint Composition

The device fingerprint is computed as:
fingerprint = SHA-256(device_id || os_version || model_name || architecture)
This identifies a unique device without relying on MAC addresses or serial numbers that may change.

7. Rollback Protection

Overview: Quantum Armor Vault prevents attackers from rolling back vault state to an older version by maintaining a state version counter both client-side and server-side. Every vault modification increments this counter, and the server rejects updates with lower version numbers.

Rollback Detection Flow

Protection Mechanism Details

  • State Version Field: 64-bit counter in vault header, incremented with every modification
  • Server-Side Tracking: max_state_version stored per vault, only increases monotonically
  • Rejection Criteria: Updates with state_version ≤ max_state_version are rejected immediately
  • Integrity: Prevents attackers from using older vault backups to overwrite newer changes
  • Limitation: Protects against overwrite attacks, not against unauthorized deletion of entire vaults
