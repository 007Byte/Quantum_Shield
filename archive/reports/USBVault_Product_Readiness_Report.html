Launch Blockers & Gaps

Critical Blockers (Must Resolve Before Launch)

1. Apple Developer Account Configuration

eas.json contains placeholder values: CONFIGURE_APPLE_ID, CONFIGURE_ASC_APP_ID, CONFIGURE_TEAM_ID. Cannot submit to App Store without these.

2. Google Play Service Account Key

eas.json references ./google-play-key.json which does not exist. Required for Play Store automated submissions.

3. Certificate Pinning — Production Pins

.env.production.example has REPLACE_WITH_PRIMARY_PIN_SHA256_BASE64 placeholders. The app's startup validation will throw and crash in production if these aren't replaced with real certificate pins.

4. Sentry Production DSN

Error tracking is disabled without a real Sentry DSN. Crashes in production will be invisible without this.

5. Production API Deployment

The Go API server and PostgreSQL database must be deployed and reachable at the configured EXPO_PUBLIC_API_URL. Without this, the app runs in demo mode only.

6. Version Number (0.1.0 → 1.0.0)

Both app.json and package.json show version 0.1.0. App stores will display this to users. Must be bumped to 1.0.0 for GA launch.
