Recommendations & Next Steps

1. Fix Test Import Paths P0

Update all existing test files to import from the new consolidated domain barrel paths (@/services/billing, @/services/messaging, etc.). Without this, the CI test suite will fail on every run. This is a ~30-minute mechanical task.

2. Add Server Recovery Package Tests P0

Write tests for internal/recovery covering SSS share generation, reconstruction at/below/above threshold, and tamper-resistant share encoding. Shamir Secret Sharing is a critical security feature — bugs here mean permanent user lockout.

3. Write App fido2Service Tests P0

The client FIDO2 service (fido2Service.ts) is completely untested. Add tests for challenge handling, assertion creation, registration flow, and error cases (credential not found, user cancelled). Mock the platform WebAuthn API.

4. Write syncService Tests P1

WebSocket sync client has no coverage. Add tests for connection lifecycle, reconnect with exponential backoff, conflict resolution on stale updates, and proper cleanup on logout. This directly affects data consistency.

5. Add usbService Tests P1

USB air-gap vault operations are a core differentiator with zero test coverage. Write unit tests for USB detection, vault export/import, and error cases (device disconnected mid-transfer). Mock the native USB layer.

6. Session Service Tests P1

Add tests for auto-lock on inactivity, background app pause/resume locking, failed biometric → PIN fallback, and force-lock on security events. These protect users from shoulder surfing and unauthorized access.

7. Expand Sharing Domain Tests P2

The new consolidated sharing/sharing.ts merges 3 services but has no tests. Add coverage for group messaging key rotation on member removal, external portal download-limit enforcement, and PIN protection on external shares.

8. Server Security Package P2

Write unit tests for internal/security including anomaly detection thresholds, brute-force classification, and dark web alert triggers. Mock external security API calls (HaveIBeenPwned, dark web feeds).

9. Emergency Access Tests P2

Emergency access flow (trusted contact key escrow) has no app-side tests. Add coverage for trust establishment, key release after delay period, and revocation. This is a high-trust flow with complex state.

10. Set Up Coverage Thresholds in CI P3

Configure Jest coverage thresholds (e.g., 70% line coverage minimum) and Go go test -coverprofile with coverage gates. Block merges that drop coverage below threshold. Target: 80% app services, 85% server packages.

11. Add Integration Tests for Billing Flows P3

Stripe webhook processing is server-tested but the full upgrade → feature-unlock → downgrade → limit-enforcement cycle has no end-to-end test. Add an integration test suite using Stripe test mode webhooks.

12. E2E Test Suite (Detox / Playwright) P3

No end-to-end UI tests exist. Consider adding Detox (mobile) or Playwright (web) for the critical user journeys: login, vault create, file upload, share, and logout. Even 5–10 happy-path E2E tests provide high confidence before releases.
