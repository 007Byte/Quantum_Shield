Identified Gaps & Missing Items

GAP-1: Production Certificate Pins (BLOCKER)

Certificate pinning is implemented with placeholder SHA-256 SPKI hashes (AAAA... patterns). Semgrep rule blocks production deployment with placeholders. Resolution: Extract real pins from production domain TLS certificate before app store submission.

GAP-2: External Cryptographic Review (RECOMMENDED)

Phase 10 AST Gate issued CONDITIONAL PASS. Internal audit is thorough (constant-time, memory safety, protocol verification) but no independent third-party review of the Rust crypto core has been conducted. Resolution: Engage a cryptography firm 4-6 weeks before GA.

GAP-3: Post-Quantum Cryptography Not in Main Path

ML-KEM-1024 and ML-DSA-87 have test harnesses and feature flags, but are NOT wired into the primary encryption/signing path. Only X25519 + Ed25519 are active. Resolution: Enable hybrid X25519 + ML-KEM-768 key exchange before quantum threat timeline (~2030).

GAP-4: Service Domain Consolidation Incomplete

4 of 7 service domains still need consolidation: Sharing (3 files), Messaging (3 files), Device (2 files), Billing (2 files). Current: 60 root files, target: ~35. Resolution: Complete consolidation to reduce maintenance burden.

GAP-5: Legal Documents Not Published

Privacy policy and Terms of Service are referenced in launch checklist but not yet drafted or published. These are mandatory for app store submission. Resolution: Draft and publish to usbvault.io domain.

GAP-6: Multipart Upload State Not Persistent

Multipart upload tracking is in-memory only. Server restart loses all in-flight upload state. Resolution: Add PostgreSQL-backed state persistence for upload tracking.

GAP-7: JWT Private Key Storage

JWT signing keys are stored base64-encoded in PostgreSQL. Production should use KMS envelope encryption. Resolution: Integrate AWS KMS or Vault for key material protection.
