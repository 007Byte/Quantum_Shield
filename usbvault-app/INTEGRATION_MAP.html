📦 Zustand Stores (State Management)

▼
🔑 authStore.ts
  • State: isAuthenticated, userId, email, subscriptionTier, isLoading, error
  • Actions: login(), register(), logout(), checkAuth(), lockVault()
  • Web Storage: localStorage (auth), sessionStorage (session)
  • Native Bridge: authService + api calls
  • Integration: Used by TopBar, Settings screens, Auth flows
SHA-256 hashing (web) SRP-6a protocol (native) Session management
🗃️ vaultStore.ts
  • State: vaults[], currentVault, files[], isLoading, error
  • Actions: loadVaults(), selectVault(), createVault(), deleteVault(), loadFiles(), addFile(), deleteFile(), renameVault(), exportVault()
  • Web Storage: IndexedDB via webStorage service (off-heap blob storage)
  • Native Bridge: api.ts calls
  • Fallback: WEB_MOCK_VAULTS and WEB_MOCK_FILES for preview mode

IndexedDB (web) Mock data fallback Audit logging on changes

⚙️ Core Services Layer

▼
🌐 API & Authentication Services
  • api.ts — HTTP client with cert pinning, retry logic, JWT auto-refresh. Endpoints: getUserInfo, listVaults, createVault, deleteVault, SRP endpoints, share management, FIDO2 device management
  • auth.ts — SRP-6a login/register/logout, isAuthenticated check, clearMasterKey for locking
  • certificatePinning.ts — SPKI certificate pinning with fail-closed security model
  • deviceIntegrity.ts — Runtime detection: jailbreak, root, debugger, emulator, Frida framework
Integrations: authStore calls auth.ts + api.ts; API client auto-attaches JWT from SecureStore
🔒 Security & Encryption Services
  • shareService.ts — X25519 sealed box file sharing. Methods: getOrCreateKeypair, shareFile, acceptShare, rejectShare, revokeShare, getOutgoing/IncomingShares, unsealFileKey
  • fido2Service.ts — WebAuthn FIDO2 device registration/authentication. Methods: registerDevice, authenticate, listDevices, removeDevice, getDeviceCount
  • appProtection.ts — Auto-lock, clipboard clearing, screenshot prevention. Hook: useAppProtection
  • forensicsService.ts — Anti-forensics cleanup. Categories: clipboard, app_cache, recent_files, session_data, temp_files, swap_pagefile, os_journals. Config: cleanOnLock, cleanOnLogout, scheduledIntervalMin
X25519 ECDH WebAuthn RAM scrubbing Auto-cleanup
💬 Messaging & Data Services
  • messageService.ts — E2E encrypted messaging. Methods: sendMessage, getConversations, getMessages, decryptMessage, markAsRead, deleteMessage/Conversation. FEAT-14: Ghost Messages (self-destruct, timers: 5s–24h)
  • auditService.ts — Activity logging. Methods: log(), getEntries(), exportLogs(), clear(). Storage: localStorage, 500-entry cap, FIFO. Actions: encrypt, decrypt, share, login, logout, etc.
  • syncService.ts — Queue-based offline-first sync with retry/backoff logic
  • webStorage.ts — IndexedDB persistence for vaults/files on web platform
⚡ User Settings & Utilities
  • settingsService.ts — User preferences. Config: biometricLockEnabled, twoFactorEnabled, autoLockTimeoutMin, ghostModeEnabled, selfDestructEnabled, keyProvider (software/hardware/hybrid), pqcEnabled, autoBackupEnabled, notificationsEnabled
  • importService.ts — Password import from 5 formats (CSV/JSON)
  • passwordService.ts — CRUD with AES-GCM encrypted localStorage
  • supportService.ts — FAQ, tickets, status page

🔐 Cryptography Layer

▼
🦀 crypto/bridge.ts (Rust FFI Bridge)
  • Key Derivation: deriveKey(password, salt, params) — Argon2id
  • AEAD Encryption: encrypt(plaintext, key, nonce), decrypt(ciphertext, key, nonce) — ChaCha20-Poly1305
  • Key Exchange: generateShareKeypair() — X25519, sealToPublicKey(plaintext, pubkey), openSealed(ciphertext, keypair)
  • SRP-6a: srpGenerateClientEphemeral(N, g), srpDeriveSession(password, salt, A, B, u)
  • Signing: sign(message, privateKey), verify(message, signature, publicKey) — Ed25519
  • Streaming: encryptStream(), decryptStream() for large files
  • Hashing: hashSha256(data)
Argon2id ChaCha20-Poly1305 X25519 ECDH Ed25519
📱 crypto/native.ts
  • Native module bindings for iOS/Android
  • Delegates to Rust bridge via React Native turbomodules

🌍 Platform Support & Fallbacks

▼
🌐 Web Platform (Browser)
  • Storage: localStorage (auth, session, settings, messages), sessionStorage (session data), IndexedDB (vaults, files via webStorage)
  • Crypto: WebCrypto API (SHA-256), Rust bridge via wasm
  • Biometrics: WebAuthn FIDO2 via fido2Service
  • Fallback Data: WEB_MOCK_VAULTS, WEB_MOCK_FILES for preview when offline or data unavailable
  • API: axios HTTP client to Go backend (with retry, JWT auto-refresh, device fingerprint binding)
📱 Native Platform (iOS/Android via Expo)
  • Storage: SecureStore (tokens, keys), async FileSystem
  • Crypto: Rust bridge via React Native turbomodules
  • Biometrics: expo-local-authentication (native fingerprint/face)
  • API: Same axios client with native cert pinning hooks
  • Device Integrity: Native jailbreak/root detection via deviceIntegrity service
🔄 Graceful Degradation
  • No API Connection: Use mock data (web), cached vaults (native)
  • No Biometrics: Fall back to password + PIN
  • No WebCrypto (legacy browser): Use simpler SHA-256 fallback
  • IndexedDB Error: Fall back to mockData, warn user
  • Secure Store Error (native): Graceful fail, redirect to login

🔒 Security Architecture Highlights

▼
🛡️ Authentication & Authorization
  • SRP-6a: Secure Remote Password protocol (prevents server compromise from exposing passwords)
  • JWT Tokens: Access + refresh token pair, auto-refresh on 401
  • Device Binding: Device fingerprint included in refresh token validation (DV-008)
  • Subscription Tiers: free, pro, enterprise (enforced at API level)
🔐 Encryption & Key Management
  • Key Derivation: Argon2id (memory-hard, resistant to GPU/ASIC attacks)
  • File Encryption: ChaCha20-Poly1305 AEAD (authenticated encryption)
  • Sharing: X25519 ECDH sealed box (per-file key exchange)
  • Master Key: Never stored; derived from password + salt on each session
  • Streaming Mode: For large files (avoid loading entire file in memory)
🚨 Defense-in-Depth
  • Certificate Pinning: SPKI pins with fail-closed enforcement (api.ts, certificatePinning.ts)
  • Device Integrity Checks: Jailbreak/root/debugger/emulator/Frida detection at runtime (deviceIntegrity.ts)
  • Auto-Lock: Vault locks after timeout (appProtection.ts, settingsService)
  • Clipboard Auto-Clear: Overwrite clipboard after copy (appProtection.ts)
  • Screenshot Prevention: Block screenshots on sensitive screens (appProtection.ts)
  • Anti-Forensics (FOR-01): RAM scrubbing, temp file cleanup, session data wipe (forensicsService.ts)
👤 User Privacy & Data Minimization
  • E2E Messaging: Messages encrypted client-side, server never sees plaintext (messageService.ts)
  • Ghost Mode (FEAT-14): Self-destructing messages with configurable timers (5s–24h) (messageService.ts)
  • Metadata Reduction: Minimal file metadata stored on server (SEC-05)
  • Audit Logging: Local-only audit log (localStorage, 500-entry cap, encrypted if applicable)
  • No Tracking: No external analytics, no device fingerprinting for ad purposes
🔄 Data Integrity & Sync
  • Offline-First Sync: Queue-based sync with exponential backoff (syncService.ts)
  • Retry Logic: API client retries network errors up to 3 times (api.ts)
  • Transactional Integrity: Vault and file operations are atomic
